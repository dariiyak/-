# Разбор `benchmark.cpp` по функциям (в порядке кода)

## Суть файла (кратко для защиты)

`benchmark.cpp` — это программа для измерения производительности сортировок на одинаковых входных данных.

Что делает:
- читает записи формата `key<TAB>value` из `stdin`;
- запускает вашу `bucket_sort` и `std::sort` на одной и той же выборке;
- повторяет каждый запуск 5 раз и считает среднее время в микросекундах;
- проверяет, что результат каждой сортировки действительно отсортирован;
- выводит итог в формате: `N BUCKET_US STD_SORT_US`.

Ниже разбор всех функций и методов из `benchmark.cpp` в том порядке, как они идут в файле.

## Константы

- `MAX_VALUE_LEN = 2048` — максимум длины `value`.
- `MAX_LINE_LEN = 4096` — максимум длины входной строки.
- `BENCH_RUNS = 5` — сколько раз повторяется каждый алгоритм для усреднения.
- `MIN_KEY = -100.0`, `MAX_KEY = 100.0` — допустимый диапазон ключа.

## `struct Record`

Одна запись данных:
- `double key`
- `char* value`

Формат записи соответствует строке `key<TAB>value`.

## `static char* clone_cstr(const char* src)`

Назначение: сделать глубокую копию C-строки.

Что делает:
1. Считает длину `src` через `std::strlen`.
2. Выделяет `new char[len + 1]`.
3. Копирует байты через `std::memcpy` вместе с `\0`.
4. Возвращает указатель на новую строку.

Используется при добавлении записей в `RecordArray`.

## `struct RecordArray`

Динамический массив `Record` с полями:
- `data`, `size`, `capacity`;
- `owns_values` — флаг владения строками `value`.

Идея `owns_values`:
- `true` — объект отвечает за удаление `value`;
- `false` — строки не удаляются (чтобы не удалить одни и те же строки дважды).

## `RecordArray::RecordArray()`

Инициализация пустого массива:
- `data = 0`, `size = 0`, `capacity = 0`, `owns_values = true`.

## `RecordArray::~RecordArray()`

Деструктор:
1. Вызывает `clear_values()`.
2. Удаляет массив `data`.

## `void RecordArray::clear_values()`

Удаляет строки `value`, только если `owns_values == true`.

Логика:
- если не владеет строками, сразу `return`;
- иначе проходит по всем элементам и делает `delete[] data[i].value`.

## `void RecordArray::reserve(int new_capacity)`

Расширяет емкость массива:
1. Если `new_capacity <= capacity`, ничего не делает.
2. Иначе выделяет новый массив `Record[new_capacity]`.
3. Копирует существующие элементы.
4. Удаляет старый массив.
5. Обновляет `data` и `capacity`.

## `void RecordArray::push_back(double key, const char* value)`

Добавляет запись в конец:
1. При необходимости расширяет буфер (`8, 16, 32, ...`).
2. Копирует строку `value` через `clone_cstr`.
3. Записывает `key` и `value` в `data[size]`.
4. Увеличивает `size`.

## `struct Bucket`

Временный динамический массив `Record` для bucket sort:
- `data`, `size`, `capacity`.

## `Bucket::Bucket()`

Создает пустой бакет (`data = 0`, `size = 0`, `capacity = 0`).

## `Bucket::~Bucket()`

Удаляет только массив `data`.

Важно: строки `value` тут не удаляются, бакет ими не владеет.

## `void Bucket::push_back(const Record& rec)`

Добавляет запись в бакет:
1. Если нужно, расширяет внутренний массив (`4, 8, 16, ...`).
2. Копирует `Record` в конец (`data[size++] = rec`).

Здесь `value` копируется как указатель, не как новая строка.

## `static void insertion_sort_bucket(Bucket& bucket)`

Сортирует один бакет методом вставок по `key`.

Коротко:
- идет слева направо;
- для каждого элемента сдвигает большие ключи вправо;
- вставляет текущий элемент в правильную позицию.

## `static void bucket_sort(Record* arr, int n)`

Основная сортировка для теста.

Шаги:
1. Если `n <= 1`, выход.
2. Число бакетов: `bucket_count = n` (минимум 2).
3. Создаются `bucket_count` бакетов.
4. Каждая запись раскладывается по индексу:
   - `normalized = (key - MIN_KEY) / (MAX_KEY - MIN_KEY)`
   - ограничение в `[0, 1]`
   - `idx = int(normalized * bucket_count)`
   - защита от выхода за правую границу.
5. Каждый бакет сортируется `insertion_sort_bucket`.
6. Элементы бакетов последовательно собираются обратно в `arr`.
7. Массив бакетов освобождается.

## `static bool check_sorted(const RecordArray& arr)`

Проверка, что массив отсортирован по неубыванию `key`.

Если встречается `arr[i-1].key > arr[i].key`, возвращает `false`, иначе `true`.

## `static bool record_less(const Record& a, const Record& b)`

Компаратор для `std::sort`:
- возвращает `a.key < b.key`.

## `static void copy_records(Record* dst, const Record* src, int n)`

Быстро копирует `n` элементов `Record` через `std::memcpy`.

Зачем нужно:
- перед каждым запуском сортировки берется одинаковая исходная копия данных.

## `static bool parse_line(const char* line, double* key_out, const char** value_out)`

Разбирает одну входную строку и валидирует её.

Что проверяет:
1. Наличие таба `\t`.
2. Корректный числовой ключ через `std::strtod`.
3. Отсутствие мусора после числа.
4. Диапазон ключа `[MIN_KEY, MAX_KEY]`.
5. Ограничение длины `value` до `MAX_VALUE_LEN`.

Если строка корректна:
- записывает `key_out` и `value_out`;
- возвращает `true`.

Иначе возвращает `false`.

## `int main()`

Точка входа бенчмарка.

Порядок работы:
1. Читает `stdin` построчно.
2. Для каждой строки вызывает `parse_line`; некорректные строки пропускает.
3. Корректные записи складывает в `source`.
4. Если данных нет, печатает только заголовок `# N BUCKET_US STD_SORT_US` и выходит.
5. Создает рабочий массив `work` такого же размера, но с `owns_values = false`.
6. Инициализирует накопители времени:
   - `bucket_total_us`
   - `std_sort_total_us`
7. В цикле `BENCH_RUNS` раз:
   - копирует `source -> work`;
   - меряет время `bucket_sort`;
   - проверяет `check_sorted`;
   - снова копирует `source -> work`;
   - меряет время `std::sort`;
   - проверяет `check_sorted`.
8. Считает среднее время (целочисленное деление на `BENCH_RUNS`).
9. Печатает результат:
   - заголовок `# N BUCKET_US STD_SORT_US`
   - строка: `<N> <bucket_avg_us> <std_sort_avg_us>`.

## Что именно сравнивает бенчмарк

Сравниваются две сортировки на одинаковом наборе записей:
- ваша `bucket_sort`;
- стандартная `std::sort`.

Обе сортировки запускаются по 5 раз, затем берется среднее время в микросекундах.
