# Лабораторная работа 4 (Prolog, DCG)

## Тема
Синтаксический разбор русских вопросов с помощью DCG (Definite Clause Grammar) и построение логической (семантической) формы запроса.

## Цель работы
1. Формально описать допустимые шаблоны вопросов на Prolog-DCG.
2. Выполнить синтаксический разбор входного списка токенов.
3. Сконструировать из разбора семантический терм, пригодный для логического вывода.

Файл с программой: `lab4/lab4.pl`.

## Идея решения (коротко)
Я разделила задачу на 2 уровня:
1. **Синтаксис**: какие последовательности токенов считаются правильным вопросом.
2. **Семантика**: какой логический терм должен получиться для каждого типа вопроса.

Вход: список токенов (слов + `?`).
Выход: терм вида `любит(...)` или `лежать(...)` с переменной в неизвестной позиции.

## Главный предикат `an_q/2`

```prolog
an_q(Tokens, X) :- phrase(question(X), Tokens).
```

Что важно понимать:
- `question(X)` — это стартовый нетерминал грамматики.
- `phrase/2` проверяет, что **весь** список `Tokens` разобран (без хвоста).
- если разбор успешен, `X` связывается с семантическим термом.
- если хотя бы одно правило не подходит, результат `false`.

## Как работает `question/1`

```prolog
question(X) --> who_q(X).
question(X) --> what_q(X).
question(X) --> where_q(X).
```

Это дизъюнкция по типам вопросов:
- вопрос на агента: «Кто ...?»
- вопрос на объект: «Что ...?»
- вопрос на локацию: «Где ...?»

Prolog пробует правила сверху вниз и использует backtracking.

## Подробный разбор каждого шаблона

### 1) `who_q/1` — вопрос «Кто ...?»

Синтаксис:
- вопросительное слово `Кто/кто`
- глагол `любит`
- объект (`шоколад`/`деньги`)
- знак вопроса

Семантика:
```prolog
X =.. [F, agent(_Y), object(Obj)]
```
Если `F = 'любит'` и `Obj = 'шоколад'`, получаем:
```prolog
X = любит(agent(_Y), object(шоколад))
```
Неизвестен именно агент, поэтому стоит переменная `_Y`.

### 2) `what_q/1` — вопрос «Что ...?»

Синтаксис:
- `Что/что`
- `любит`
- имя субъекта (`Даша`)
- `?`

Семантика:
```prolog
X =.. [F, agent(Name), object(_Y)]
```
Неизвестен объект (`object(_Y)`), потому что вопрос про «что».

### 3) `where_q/1` — вопрос «Где ...?»

Синтаксис:
- `Где/где`
- `лежит/лежат`
- объект
- `?`

Семантика:
```prolog
X =.. [F, object(Obj), loc(_L)]
```
Неизвестна локация (`loc(_L)`), потому что вопрос про «где».

## Лексика и нормализация

### Вопросительные слова
- `who` распознает `Кто` и `кто`.
- `what` распознает `Что` и `что`.
- `where` распознает `Где` и `где`.

### Глаголы
- `love_verb('любит') --> ['любит']`.
- `lie_verb('лежать') --> ['лежит']` и `['лежат']`.

Ключевая идея: словоформы `лежит/лежат` нормализуются в одно семантическое имя предиката `лежать`.

### Лексические ограничения
- объекты только: `шоколад`, `деньги`.
- субъект в `what_q` только: `Даша`.

Это осознанное ограничение мини-грамматики.

## Что делает `=..` (univ)

`=..` превращает список в терм (или наоборот).

Пример:
```prolog
X =.. ['любит', agent(Даша), object(_)].
```
Эквивалентно:
```prolog
X = любит(agent(Даша), object(_)).
```

Зачем это здесь:
- можно сначала распознать глагол в `F`, потом собрать итоговый предикат динамически.

## Пошаговая трассировка разбора (для защиты)

Пример:
```prolog
?- an_q(['Кто','любит','шоколад','?'], X).
```

Шаги:
1. Вызов `phrase(question(X), Tokens)`.
2. `question(X)` пробует `who_q(X)`.
3. `who` съедает токен `'Кто'`.
4. `love_verb(F)` съедает `'любит'`, получает `F = 'любит'`.
5. `obj_noun(Obj)` съедает `'шоколад'`, получает `Obj = 'шоколад'`.
6. `['?']` съедает `'?'`.
7. В `{ ... }` строится `X = любит(agent(_), object(шоколад))`.
8. Остаток токенов пустой, разбор успешен.

### Некорректные запросы
```prolog
?- an_q(['Кто','любит','Даша','?'], X).
false.

?- an_q(['Что','лежит','Даша','?'], X).
false.

?- an_q(['Где','любит','шоколад','?'], X).
false.
```

Причины `false`:
- токены не соответствуют нужному шаблону;
- слово не входит в нужный нетерминал (например, имя вместо объекта);
- выбран «не тот» глагол для структуры вопроса.

## Почему решение корректно
1. Каждый тип вопроса задается отдельным строгим шаблоном.
2. `phrase/2` требует полного покрытия входа, лишние токены недопустимы.
3. Семантика строится только после успешного синтаксического распознавания.
4. Неизвестная часть вопроса кодируется переменной именно в нужном аргументе (`agent/object/loc`).

## Частые вопросы преподавателя и ответы

1. **Почему вход списком токенов, а не строкой?**
Потому что DCG в базовом виде удобнее работает со списками символов/токенов; токенизация вынесена отдельно, это упрощает грамматику.

2. **Почему в `where_q` используется `лежать`, а не `лежит/лежат`?**
Это нормализация словоформ к одной лемме. Так семантическая база чище и единообразнее.

3. **Что означает переменная в результате (`agent(_)`)?**
Это неизвестная величина, которую пользователь спрашивает. Тип вопроса определяет, где стоит переменная.

4. **Зачем `=..`, если можно написать терм напрямую?**
`=..` полезен, когда имя предиката приходит из лексики (`F`) и нужно собрать терм динамически.

5. **Почему некоторые фразы отвергаются?**
Потому что грамматика намеренно ограниченная: принимает только описанные конструкции и словарь.
