# Лабораторная 2

## Суть задания (твой вариант)
Дана логическая задача:
- 4 человека: Воронов, Павлов, Левицкий, Сахаров;
- 4 занятия: танцор, художник, певец, писатель;
- каждому человеку соответствует ровно одно занятие.

Нужно определить, кто кем является, используя условия из текста задачи.

Файл с решением: `lab2.pl`.

## Что делает программа
Программа перебирает все возможные назначения ролей и оставляет только те, которые удовлетворяют ограничениям из условия.

Подход классический для Prolog:
1. Сначала генерируются кандидаты (`permutation/2`).
2. Потом накладываются логические ограничения (`\=`).
3. Оставшийся вариант и есть решение.

## Подробная логика решения (по шагам)

### Шаг 1. Формируем пространство всех вариантов
Берём 4 роли:
- `dancer` (танцор)
- `artist` (художник)
- `singer` (певец)
- `writer` (писатель)

И распределяем их по 4 людям:
- Воронов -> `RV`
- Павлов -> `RP`
- Левицкий -> `RL`
- Сахаров -> `RS`

Через `permutation(Roles, [RV, RP, RL, RS])` получаем все возможные взаимно-однозначные назначения (всего 4! = 24 варианта).

Важно: за счёт `permutation/2` одна и та же роль не может повторяться у двух людей.

### Шаг 2. Переводим текст задачи в ограничения
Дальше код не «угадывает», а просто отбрасывает неподходящие варианты.

Ограничения в программе:
- `RV \= singer` и `RL \= singer`
  - Воронов и Левицкий не певец.
- `RP \= writer`
  - Павлов не писатель.
- `RP \= artist`
  - Павлов не художник.
- `RS \= writer`
  - Сахаров не писатель.
- `RV \= writer`
  - Воронов не писатель.
- `RV \= artist`
  - Воронов не художник.

Каждая строка с `\=` убирает часть из 24 комбинаций.

### Шаг 3. Backtracking Prolog
Prolog работает так:
1. берёт первую перестановку;
2. проверяет ограничения сверху вниз;
3. если хотя бы одно условие нарушено, откатывается (backtracking) и берёт следующую перестановку;
4. когда найдён вариант, где все условия истинны, возвращает ответ.

Если нажать `;`, Prolog попробует найти следующий допустимый вариант (если он есть).

### Шаг 4. Формат результата
- `solve(A)` возвращает ответ в виде списка:
  - `A = [role(voronov, ...), role(pavlov, ...), role(levitsky, ...), role(sakharov, ...)]`
- `solution(Voronov, Pavlov, Levitsky, Sakharov)` возвращает тот же ответ, но удобнее для чтения:
  - отдельно роль каждого человека в своём аргументе.

### Что важно концептуально
- Это не императивный «алгоритм с циклами», а декларативная модель:
  - задаём множество вариантов,
  - задаём систему ограничений,
  - Prolog сам находит совместимые решения.

## Разбор кода по предикатам

### `:- use_module(library(lists)).`
Подключает библиотеку списков, чтобы использовать `permutation/2`.

### Что такое `permutation/2` и запись `имя/арность`
- `permutation/2` — стандартный предикат Prolog из `library(lists)`, который работает с перестановками списка.
- Общий вид:
  - `permutation(List, PermutedList)`
- Смысл:
  - `PermutedList` содержит те же элементы, что и `List`, но в другом (или том же) порядке.
- В задаче:
  - `permutation([dancer, artist, singer, writer], [RV, RP, RL, RS])`
  - это означает «распредели 4 разные роли по 4 людям всеми возможными способами».

Что означает запись `permutation/2`:
- В Prolog часто пишут `имя/число`.
- Число после `/` — это арность предиката, то есть количество аргументов.
- Поэтому:
  - `permutation/2` -> предикат `permutation` с 2 аргументами.
  - `solve/1` -> предикат `solve` с 1 аргументом.
  - `solution/4` -> предикат `solution` с 4 аргументами.

Мини-пример:
```prolog
?- permutation([a,b,c], P).
P = [a, b, c] ;
P = [a, c, b] ;
P = [b, a, c] ;
...
```

### `solve(Assignments)`
Главный предикат решения.

Что происходит внутри:
- задаётся список ролей: `[dancer, artist, singer, writer]`;
- через `permutation/2` роли распределяются между:
  - `RV` -> Воронов,
  - `RP` -> Павлов,
  - `RL` -> Левицкий,
  - `RS` -> Сахаров;
- формируется итоговый список `Assignments` в виде `role(человек, роль)`;
- применяются ограничения:
  - `RV \= singer` -> Воронов не певец;
  - `RL \= singer` -> Левицкий не певец;
  - `RP \= writer` -> Павлов не писатель;
  - `RP \= artist` -> Павлов не художник;
  - `RS \= writer` -> Сахаров не писатель;
  - `RV \= writer` -> Воронов не писатель;
  - `RV \= artist` -> Воронов не художник.

### `solution(Voronov, Pavlov, Levitsky, Sakharov)`
Удобная обёртка над `solve/1`.

Позволяет получить ответ в формате:
- роль Воронова,
- роль Павлова,
- роль Левицкого,
- роль Сахарова.

## Почему решение корректно
- `permutation/2` гарантирует, что каждая из 4 ролей назначается ровно одному человеку.
- Ограничения отсекают невозможные варианты.
- Когда все ограничения выполнены, получаем допустимое решение задачи.
