### Задание 1. Ряд Тейлора (вариант 28)
Вариант 28:
- функция: `f(x) = -ln|2 sin(x/2)|`;
- интервал: `[pi/5; 6pi/5]`;
- ряд: `sum(cos(n*x)/n)`;
- 10 равномерных точек.

Для каждой точки я считаю три значения:
- `Builtin` (через встроенные функции F#),
- `Dumb Taylor` - Используя наивный способ вычисления ряда Тейлора, где каждый член ряда вычисляется по формуле,
- `Smart Taylor` - Используя умный способ вычисления ряда Тейлора, где каждый следующий член вычисляется на основе предыдущего..

#### Ключевые функции и что они делают

1. `whileLoop condition step state`  
Это функциональный аналог `while`.  
Я передаю в него условие продолжения, шаг итерации и текущее состояние.  
Эта функция используется в обоих способах суммирования ряда, поэтому нет дублирования циклов.

2. `linspace left right count`  
Создает равномерную сетку точек на интервале.  
В лабораторной используется для 10 значений `x`, чтобы корректно заполнить таблицу.

3. `targetFunction x`  
Считает builtin-значение функции:
`-Math.Log(Math.Abs(2.0 * Math.Sin(x / 2.0)))`.  
Это эталон, с которым сравниваются оба Тейлора.

4. `taylorTerm x n`  
Считает один член ряда `cos(n*x)/n`.  
Это базовая формула варианта.

5. `dumbTaylor x`  
На каждой итерации заново вычисляет член `cos(n*x)/n`, добавляет к сумме и увеличивает `n`.  
Остановка по условию `|term| < eps` или по лимиту `maxTerms`.  
Возвращает:
- сумму ряда (`Value`),
- число использованных членов (`Terms`),
- флаг достижения лимита (`ReachedLimit`).

6. `smartTaylor x`  
Считает тот же ряд, но без повторного пересчета `cos(n*x)` с нуля.  
Используется рекуррентная формула:
```fsharp
cos((n+1)x) = 2*cos(x)*cos(nx) - cos((n-1)x)
```
Это главное ускорение задания 1: меньше дорогих вызовов `Math.Cos`.

7. `printTask1()`  
Оркестратор всего первого задания: задает параметры, строит точки, считает builtin/smart/dumb, печатает таблицу и предупреждение при `maxTerms`.

#### Почему задание 1 сделано качественно
- Есть сравнение с эталонным builtin-значением.
- Видна и точность, и вычислительная цена (число членов).
- Реализованы обе требуемые схемы ряда: наивная и оптимизированная.

### Задание 2. Численное решение уравнений (варианты 28, 1, 2)
Во второй части для трех уравнений (28, 1, 2) применяются три метода:
- дихотомия,
- простые итерации,
- Ньютон.

Итог: 9 решений в одной таблице.

#### Ключевые функции и что они делают

1. `dichotomy f left right eps maxIter`  
Метод половинного деления:
- берет середину интервала;
- выбирает половину, где сохраняется корень;
- повторяет до точности `eps` или `maxIter`.  
Плюс метода: надежность при корректном интервале.

2. `fixedPoint phi x0 eps maxIter`  
Общий метод итераций:
`x_{k+1} = phi(x_k)`.  
Остановка по `|x_{k+1} - x_k| < eps`.  
Это универсальная итерационная рамка.

3. `newton f df x0 eps maxIter`  
Реализован как частный случай `fixedPoint`:
```fsharp
let newton f df x0 eps maxIter =
    fixedPoint (fun x -> x - f x / df x) x0 eps maxIter
```
Это важная архитектурная идея: нет копипаста цикла, один общий механизм сходимости.

4. `EquationData`  
Описывает уравнение как данные:
- номер,
- текст,
- `F`,
- `DF`,
- интервал `[Left; Right]`,
- `Lambda` для метода итераций.  
Благодаря этому алгоритмы универсальны, а добавление нового уравнения делается без переписывания методов.

5. `printTask2()`  
Оркестратор второго задания:
- проходит по списку уравнений 28, 1, 2;
- для каждого вызывает три метода;
- печатает таблицу с корнем, `f(root)` и числом итераций.

#### Почему задание 2 сделано качественно
- Все три метода реализованы в едином стиле и легко сравниваются.
- Ньютона построен на общей итерационной функции, как требуется в условии.
- Вывод `f(root)` позволяет сразу проверить качество найденного корня.
