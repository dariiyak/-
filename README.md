# Разбор `lab1.cpp` по функциям (в порядке кода)

## Суть файла (кратко для защиты)

`lab1.cpp` — это программа, которая читает записи формата `key<TAB>value`, сортирует их по ключу и печатает результат.

Что реализовано:
- чтение входа из `stdin` построчно;
- валидация строк (формат, число, диапазон ключа `[-100, 100]`);
- хранение записей в собственном динамическом массиве `RecordArray`;
- сортировка методом `bucket sort` с `insertion sort` внутри каждого бакета;
- вывод отсортированных записей в `stdout` с точностью ключа до 6 знаков.

Ниже объяснение каждой функции и метода ровно в том порядке, как они идут в файле.

## Константы

- `kMinKey = -100.0`, `kMaxKey = 100.0` — допустимый диапазон ключа.
- `kMaxValueLen = 2048` — максимальная длина `value`.
- `kMaxLineLen = 4096` — максимальная длина читаемой строки.

## `struct Record`

`Record` — минимальная единица данных, одна строка входа после разбора.

Поля:
- `double key` — числовой ключ, по нему выполняется сортировка;
- `char* value` — текстовая часть строки (то, что было после `\t`).

Как это связано с форматом входа:
- строка `12.5<TAB>hello` превращается в
  - `key = 12.5`
  - `value = "hello"`.

Важно про память:
- `Record` сам по себе не выделяет и не освобождает память;
- память для `value` выделяется в `RecordArray::push_back`;
- освобождается в `RecordArray::~RecordArray`.

## `RecordArray` (понятно и коротко)

`RecordArray` — это самодельный динамический массив, аналог `std::vector<Record>`.

Он хранит:
- `data` — указатель на массив `Record`;
- `size` — сколько записей реально лежит;
- `capacity` — сколько записей влезает без расширения.

Главная идея: при чтении входа записи добавляются по одной, поэтому массив должен уметь расти.

## `RecordArray::RecordArray()`

Стартовое пустое состояние:
- `data = 0` (память под массив еще не выделена),
- `size = 0`,
- `capacity = 0`.

## `void RecordArray::reserve(int new_capacity)`

Эта функция расширяет вместимость.

Если новый размер больше текущего:
1. Выделяется новый массив `Record[new_capacity]`.
2. Все старые элементы копируются в новый массив.
3. Старый массив удаляется.
4. `data` начинает указывать на новый массив, `capacity` обновляется.

Зачем это нужно: чтобы не выделять память на каждый `push_back`, а расширяться реже и крупнее.

## `void RecordArray::push_back(double key, const char* value)`

Назначение: добавить новую запись `(key, value)` в конец динамического массива.

Логика работы:
1. Проверка места.
Если `size == capacity`, массив расширяется через `reserve` (рост емкости: `8, 16, 32, ...`).

2. Копирование строки `value`.
Считается длина строки, выделяется новый буфер `char[]` длиной `value_len + 1`, затем в него копируется содержимое вместе с завершающим `'\0'`.

3. Запись данных в новую ячейку.
В `data[size]` сохраняются:
- ключ `key`;
- указатель на выделенную копию строки.

4. Фиксация добавления.
`size` увеличивается на 1.

Почему это важно:
- исходный `value` приходит из временного буфера `line` в `main`;
- этот буфер перезаписывается при чтении следующей строки;
- поэтому нужна отдельная (глубокая) копия строки для каждой записи.

## `RecordArray::~RecordArray()`

Кто владел памятью, тот ее и освобождает:
1. Проходит по всем `size` записям и делает `delete[] data[i].value`.
2. Делает `delete[] data` для массива `Record`.

Итог: утечек по данным `RecordArray` не остается.

## `Bucket::Bucket()`

Конструктор одного бакета.
Инициализирует пустой динамический массив:
- `data = 0`, `size = 0`, `capacity = 0`.

## `Bucket::~Bucket()`

Освобождает массив записей бакета `data`.

Важно: строки `value` тут не удаляются, потому что бакет хранит копии `Record` с теми же указателями на строки, владельцем которых остается `RecordArray`.

## `void Bucket::push_back(const Record& rec)`

Назначение: добавить одну запись в конкретный бакет во время `bucket_sort`.

Логика работы:
1. Проверка свободного места.
Если `size == capacity`, бакет расширяется:
- выбирается новая емкость (`4`, затем `8`, `16`, ...);
- выделяется новый массив `Record`;
- старые элементы копируются в новый массив;
- старый массив удаляется;
- обновляются `data` и `capacity`.

2. Добавление элемента.
После гарантии места выполняется запись:
- `data[size] = rec`;
- затем `size` увеличивается.

Что именно копируется:
- копируется объект `Record` как пара `(key, value)`;
- `key` копируется по значению;
- `value` копируется как указатель `char*` (адрес строки), без создания новой строки.

Почему это корректно:
- бакет не владеет строками `value`;
- владельцем строк остается `RecordArray`;
- поэтому `Bucket::~Bucket()` удаляет только массив `Record`, но не `value`.

## `void insertion_sort_bucket(Bucket& bucket)`

Назначение: отсортировать записи внутри одного бакета по возрастанию `key`.

Как работает:
1. Считаем, что левая часть бакета уже отсортирована.
2. Берем следующий элемент `current`.
3. Сдвигаем вправо все элементы слева, у которых `key > current.key`.
4. Вставляем `current` в освободившуюся позицию.

Почему этот метод выбран:
- бакеты обычно маленькие;
- insertion sort проста и быстра на небольших массивах.

Сложность для бакета размера `k`:
- `O(k)` в лучшем случае;
- `O(k^2)` в худшем.

## `void bucket_sort(Record* records, int n)`

Главная сортировка.

Что делает по шагам:
1. Если `n <= 1`, сразу выход.
2. Выбирает число бакетов: `bucket_count = n` (минимум 2).
3. Создает массив бакетов `new Bucket[bucket_count]`.
4. Для каждой записи считает номер бакета:
   - нормализует ключ в `[0, 1]`:
     `normalized = (key - kMinKey) / (kMaxKey - kMinKey)`
   - ограничивает диапазон (clamp)
   - переводит в индекс `idx`
5. Кладет запись в нужный бакет.
6. Для каждого бакета:
   - сортирует его `insertion_sort_bucket`
   - последовательно копирует элементы обратно в `records`.
7. Освобождает массив бакетов.

## `int main()`

Точка входа программы.

Что делает по порядку:
1. Ускоряет ввод/вывод:
   - `std::ios::sync_with_stdio(false)`
   - `std::cin.tie(nullptr)`
2. Создает контейнер `RecordArray records`.
3. Читает вход построчно в `line`.
4. Для каждой строки:
   - пропускает пустые;
   - ищет `\t`; если нет — пропускает строку;
   - разделяет строку на `key_part` и `value_part`;
   - обрезает `value_part` до 2048 символов;
   - парсит ключ через `std::strtod`;
   - проверяет, что ключ корректный и в диапазоне `[-100, 100]`;
   - добавляет запись в `records` через `push_back`.
5. Вызывает `bucket_sort(records.data, records.size)`.
6. Настраивает вывод `fixed` и `precision(6)`.
7. Печатает все записи в формате:
   `key\tvalue\n`.
8. Возвращает `0`.

## Кратко по сложности

- Средний случай bucket sort: близко к `O(n)` при равномерном распределении ключей.
- Худший случай: `O(n^2)` (если почти все элементы попадут в один бакет).
- Дополнительная память: `O(n)`.
